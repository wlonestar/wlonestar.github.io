使用nginx托管hugo博客

2021-07-09
2分钟阅读时长
过程记录

这几天一直在折腾云服务器，为了避免备案，我买了一台香港的云服务器。想着把博客转移到服务器上，因为 GitHub Pages 太慢了。

可能是图片比较多，使用的那个主题比较耗资源，顺便换个主题，哪知道配置nginx就花了我两天时间，现在终于配置好了，博客顺利转移过来了，记录一下，防止忘了。

本地 hugo 博客生成

  1. 生成站点
hugo new site blog
  1. 编辑配置文件
--blog
 |-archetypes
 |-content
 |-layouts
 |-static
 |-themes
 |-config.toml

打开目录下的 config.toml 文件

baseURL = "/"
languageCode = "en-us"
title = "我的博客"
#指定主题
theme = "hyde"
[params]
#博客简介
description = "个人博客"
#博客主题颜色
themeColor = "theme-base-08"
  1. 选择主题

这里选择了一个简洁的 hype 主题做示范，更多主题可以去官网下载 themes.gohugo.io

git clone https://github.com/spf13/hyde.git themes/hype
  1. 创建文章
hugo new test.md

这样在 content 目录下新建了一个 test.md 文件，打开写入一些内容

---
date: "2019-05-23"
title: "第一篇文章"
---

### 你啊

2333

​```
print('hello world')
​```
  1. 本地预览
hugo server

这样在本地 1313 端口启动了这个项目

浏览器访问 localhost:1313 即可访问站点

服务器运行环境配置

  1. 创建 git 仓库

创建一个 blog 目录

~$ mkdir blog
~$ cd blog
~/blog$ git init --bare && cd hooks && touch post-receive
~/blog/hooks$ vim post-receive

配置钩子文件，第一行根据终端修改

#!/bin/zsh
GIT_REPO=/home/your_user/blog
TMP_GIT_CLONE=/tmp/blog
NGINX_HTML=/usr/share/nginx/html
rm -rf ${TMP_GIT_CLONE}
git clone $GIT_REPO $TMP_GIT_CLONE
rm -rf ${NGINX_HTML}/*
cp -rf ${TMP_GIT_CLONE}/* ${NGINX_HTML}

本地站点打包上传服务器

  1. 生成 public 目录

在博客目录下，使用 hugo 命令，生成 public 目录

$ pwd
../blog
$ hugo
$ cd public
$ pwd
../blog/public
  1. push 到服务器
# 进入 public 目录
git init
git add .
git commit -m "message"
git remote add origin user@your_server_ip:/home/your_user/your_blog
git push origin master

这样生成的静态资源文件全部上传到了服务器上

配置 nignx

  1. 添加 ssl 证书

在云服务器控制台申请免费 ssl 证书，下载解压找到 nginx 服务器的证书

包括证书文件和私钥文件，上传到服务器，建立目录保存，配置文件里改为所在的地址

  1. 自动配置 ssl
sudo apt-get install python3-certbot-nginx
sudo ufw allow 'Nginx Full'
sudo certbot --nginx -d your_domain_name

之后按照提示一路走下去，就配置成功了

  1. 修改 nginx 配置文件
sudo vim /etc/nginx/conf.d/default.conf

按照自己站点情况修改

server {    
	listen 80;    
	listen 443 ssl;    
	ssl on;    
	ssl_certificate /home/ubuntu/.ssl/1_blog.xyz_bundle.crt;    
	ssl_certificate_key /home/ubuntu/.ssl/2_blog.xyz.key;    
	ssl_session_timeout 5m;    
	ssl_protocols TLSv1 TLSv1.1 TLSv1.2;    
	ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:HIGH:!aNULL:!MD5:!RC4:!DHE;    
	ssl_prefer_server_ciphers on;    
	server_name blog.wangjialei.xyz;    
	location / {        
		root /usr/share/nginx/html;        
		index index.html;    
	}
}
server {    
	listen 80;    
	server_name blog.wangjialei.xyz;    
	rewrite ^(.*) https://$server_name$1 permanent;
}
  1. 重启 nginx 服务，使修改生效
sudo systemctl restart nginx.service

访问域名就可以看到部署效果了
