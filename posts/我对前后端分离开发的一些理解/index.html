我对前后端分离开发的一些理解

2021-08-23
1分钟阅读时长
知识整理

这个其实算是一种开发模式，前端和后端独立开发，独立部署，通过 json 传输数据。

无论是 JSP，还是模板引擎，都是严重依赖于后端的，jsp 里面甚至还有 Java 代码，前端一般都是后端程序员兼任。在以前那个网页还比较简单的时代确实没问题，但随着的网站页面越来越复杂，用户的需求越来越高，前端逐渐独立出来，前端也开始工程化。后端可以专注于业务逻辑本身，前端则重点放在页面开发上。

MVC （Model–view–controller） 模式，即模型-视图-控制器。模型是用于封装数据的载体，视图关注页面的展现，一般使用到的就是上面提到的 JSP 和模板引擎这种，控制器控制用户的各种请求。

图片出处：2.2. Overview of Spring MVC Architecture — TERASOLUNA Server Framework for Java (5.x) Development Guideline 5.2.1.RELEASE documentation (terasolunaorg.github.io)

为了解决这个问题，Roy Thomas Fielding 博士提出 REST 设计风格 （Representational State Transfer）

Fielding Dissertation: CHAPTER 5: Representational State Transfer (REST) (uci.edu)

用我的话简单概括就是，将客户端和服务器端的关注点分离。将用户界面所关注的逻辑和数据存储所关注的逻辑分离开，统一接口，以资源为基础，每个资源都可以通过 URL 访问到。

就像我在这篇 后端接口返回结果统一封装 文章里写的那样，前后端数据格式统一为：

{
    "code": xxx,
    "msg": "xxx",
    "data": xxxx
}

这样 SpringBoot 的 controller 只需要对客户端的请求传输相应的数据即可，MVC 中的 V 不归 SpringBoot 管了，而是交给前端来做。
