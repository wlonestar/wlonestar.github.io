记录一次做web开发的经历

2021-05-22
1分钟阅读时长
日志随笔

现在坐在图书馆写这篇文章时，再回想这一个月做项目的过程，有很多感慨，可能写得比较随意，权当做个总结。

学期初就听说这学期学院有个小比赛，内容是做个小网站。当时我肯定想参加的，虽然什么都不太懂，虽然寒假在家没有好好学。真正到项目内容公布，开始着手做的时候，又没了那种热情。看到羊哥那个长到夸张的 Java 后端学习路线，我为自己的天真感到好笑。

我连 Java 的并发多线程都没看完，就想做个小网站出来？本来都不想做了，但算是不能让我的两个队友（一个前端，一个机器学习）失望吧，就硬着头皮做了下去。

在网上看了好多资料，本来准备用 SSM 框架来构建我们这个项目的，奈何我连 spring 都基本没接触，看到那一堆的xml配置文件更是头大，随便找一个 SSM 的视频教程都有几十集，加上五月份还有很多课，怎么可能全部看完，看完还不一定自己会写。

其实这时我是真的想放弃了，感觉自己太不自量力了，加上同时我还参加了学校的一个微电影大赛，作为后期，花了大量时间在剪辑上（这个可以再单独写一篇文章hhh），项目基本没做。那段时间真的挺黑暗的，算是一直熬夜剪片子，白天上课没精神，没时间学习，项目又不想做。

终于微电影大赛作品交上去了，我可以做项目了。算是因为某个契机转变了心态吧，我想着时间也不多了，要不就以做出来为目标，选择当前比较容易构建 web 项目的 spring boot 框架，直接学习 spring boot 算了。我找了一个比较完整的视频教程，跟着做，遇到讲源码什么的地方就直接跳过，因为时间不够了，没必要去了解底层是怎么实现的，先把项目搭起来再说。因为有之前学 SSM 的一点点基础吧，说到 MVC 三层模型的时候不会陌生，入门算是差不多了（至少知道一个 spring boot web 项目需要哪些部件）。

这次比赛要我们做一个用户登录注册发布留言的小网站，那我就先看人家的登录注册是怎么写的，反正我什么也不会（唉）。就登录注册我就看了三四个开源项目，发现他们写的或多或少和我想要的有点出入，不是这里不对，就是那里出了点小问题，只能再找有详细教程的项目，但可惜的是基本没有这样的项目，或者教程很简略。这段时间我也很焦虑，拍给我的队友拖后腿，毕竟他们已经做的差不多了，前端的页面基本上写好了，机器学习部分的算法也写好了，就我这个后端框架都没搭起来（唉）。

那几天我也就是反复地看那几个开源项目的源码，试图从中获得一些灵感，唉，还真有一点灵感了，大概是离最后的汇报阶段还有两天的时候，我新建了一个工程，按照我认为正确的顺序配置好，连接数据库，写代码，大概就半个下午，还真的可以跑了！

当时我的表情和这个差不多哈哈哈，不敢相信我可以做到，心情顿时好了不少，也没那么慌了。

然后就是和前端撕逼，不是，整合前端页面了，因为我用的是 spring boot 官方推荐的 thyme leaf 模板引擎，和前端沟通过程比较曲折，就不细说了，反正最终对接上了，我也是在和前端进行亲切而友好的沟通的过程中了解到 JavaScript 和后端控制器什么的能相互影响（怪不得 node.js 能写全栈，那要我这个 Java 后端干什么，前端一个人就能把活干了哈哈哈）。

然后算是这次这个项目比较遗憾的一个地方了，我们还要实现对留言中的高频词进行词云那种形式展现出来，但是我们的机器学习的队友对 Java 不熟，只能用 python 写，用 Java 调用 python 脚本，但是这个脚本用到了一些第三方库，调用的时候有点小问题，我也查阅了好多资料，没有找到合适的解决方案，只能放弃了。

向六位评委（其实就是大三大四的学长学姐）汇报的时候，我太紧张了，说话都说不顺，还是见识的少了，没办法，可能因为这个给评委留下了不好的印象（唉）。最终评委的提问也都回答上了，算还行吧，结果也只能算还行，二等奖（九组八个奖）😂

现在项目已经开源，地址 wjl-lab/spring-board: 基于 springboot 的简单登录注册网站

不指望有人看，有需要的可以作为参考，其实还有很多小问题，但基本触及我的知识盲点。

如果有时间的话，可以再写个教程，最近应该没时间了，马上就期末考试了，这学期过的太快了，事情也很多，博客四月都没写。

以上。
